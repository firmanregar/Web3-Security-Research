# Security Policy

## Reporting Security Issues

If you discover a potential vulnerability related to any research, methodology, or tooling shared in this repository, please report it responsibly.

Security issues can be reported by contacting:

Firman Siregar
Smart Contract Security Researcher

Please include the following information when reporting an issue:

* description of the vulnerability
* steps to reproduce the issue
* potential impact
* suggested mitigation (if available)

---

## Responsible Disclosure

This repository promotes responsible security research and responsible disclosure practices.

Researchers are encouraged to:

* report vulnerabilities to affected protocol teams privately
* allow reasonable time for remediation before public disclosure
* avoid exploiting vulnerabilities for malicious purposes

---

## Scope

This repository primarily contains:

* smart contract security research
* vulnerability pattern analysis
* AI-assisted auditing methodologies
* educational security materials

Any vulnerabilities discovered in external protocols referenced in this repository should be reported directly to the respective protocol teams or their security programs.
